Kubernetes Workload Primitives

Offline notes derived from official Kubernetes documentation.

Kubernetes manages applications through workload resources such as Deployments, StatefulSets, DaemonSets, and Jobs. For AI infrastructure, these primitives matter because GPU-serving systems, controllers, storage helpers, exporters, and batch-style tasks all map to different Kubernetes workload patterns.

Deployments are useful for scalable stateless services like inference APIs. StatefulSets are used when the workload needs stable identities or storage attachments. DaemonSets are a natural fit for node-level agents such as GPU plugins or monitoring components. Jobs are appropriate for finite batch tasks like data preprocessing or one-off cluster operations.

Why It Matters For AI Infrastructure

Kubernetes is attractive when the platform needs repeatable deployment, operators, device plugins, and service lifecycle management. It is often chosen over pure HPC scheduling when the environment must run both infrastructure components and model-serving workloads in a container-native way.
